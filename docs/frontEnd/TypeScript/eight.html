WilliamFocus 2021-05-20 TypeScript

# 1、类型守卫

  • 类型守卫的作用在于触发类型缩小，还可以用来区分类型集合中的不同成员；
  • 常用的类型守卫包括 switch、字面量恒等、typeof、instanceof、in 操作符和自定义类型守卫这几种；
// 自定义类型守卫：   =>   即自定义类型判断函数
const isDog = function (animal: Dog | Cat): animal is Dog {
  return 'wang' in animal
}
const getName = (animal: Dog | Cat) => {
  if (isDog(animal)) {
    return animal.wang
  }
}

// 在 函数 或者 定时器函数 中类型守卫可能会失效
let x: string | undefined
if (x) {
  x.trim()
  const a = () => {
    x.trim() // 对象可能为“未定义”。ts(2532)
  }
}

# 2、类型兼容性

  • **子类型兼容：**所有的子类型与它的父类型都兼容；子类型组成的联合类型也可以兼容它们父类型组成的联合类型；
  • **结构类型兼容：**拥有相同类型的属性、方法的接口类型或类类型的结构一致，则它们都是互相兼容的；同时也能像继承一样也能兼容扩展后的类型；
  • 对象字面的 freshness 特性：当使用对象字面创建的子类型满足结构类型兼容时，若该对象字面量没有被变量接收，直接赋值给同结构父类型时，会触发 freshness 状态严格检测，必须完全一致才可以赋值；（解决：变量接收或类型断言）
  • **类的类型兼容：**可以完全忽略构造函数及静态属性和方法是否兼容，只需比较实例的属性和方法是否兼容即可，而私有、受保护的属性和方法，则仅当这些属性和方法源自同一个类才兼容。
  • 函数类型兼容性：——变型
    • **协变：**子父类型的类型转换保持一致；函数返回值类型是协变的；
    • **逆变：**与协变相反；函数参数类型是逆变的；
    • **双向协变：**既是协变也是逆变；
    • **不变：**不变即只要是不完全一样的类型，它们一定是不兼容的； 参数个数少的兼容个数多；可选参数可以兼容剩余参数、不可选参数

# 扩充模块类型：

// person.ts
export class Person {}

// index.ts
import { Person } from './person'
declare module './person' {
  // 扩展模块的声明
  interface Person {
    greet: () => void
  }
}

Person.prototype.greet = () => {
  console.log('Hi!')
}

# 扩充全局类型：

declare global {
  interface Array<T extends unknown> {
    getLen(): number // 声明了全局的 Array 对象有一个 getLen 方法
  }
}
Array.prototype.getLen = function () {
  // 为 Array 对象实现 getLen 方法时，TS不会报错
  return this.length
}

# 类型增强：

在 TypeScript 中，就像在 ECMAScript 2015 中一样，任何包含顶级导入或导出的文件都被视为模块(module)，所以不要在定义全局类型声明文件 global.d.ts 内添加顶层的 import 或者 export 语句；

相反，没有任何顶级导入或导出声明的文件将被视为脚本(script)，其内容可在全局范围内使用（对 module 也是可见的），因此不要定义过多全局类型，避免全局命名污染；

可以显式添加描述信息，避免类型全局可见性被破坏；
