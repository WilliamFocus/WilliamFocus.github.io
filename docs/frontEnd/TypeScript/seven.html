七、TS工具类型

WilliamFocus 2021-05-15 TypeScript

——本质就是自定义的复杂类型构造器（确切地讲是泛型 ⇒ 即类型中的函数）

# 一、官方内置类型

（——官方轮子）

# 1.操作接口类型

- Partial<T>：可以将类型 T 所有必要属性转变成可选属性； : ⇒ ?:
- Required<T>：将类型 T 可选属性去除且变成必须属性； : ⇒ -?:
- Readonly<T>：可以将给定类型的所有属性设为只读，不可以被重新赋值

- pick<T，K extends keyof T>：从给定的类型中选取出指定的键值，组成一个新的类型；
- Omit<T，K extends keyof any>：返回去除指定的键值之后返回的新类型，与 Pick 类型相反 ⇒ Pick<T, Exclude<keyof T, K>>

# 2.联合类型

- Exclude<T, U>：** _⇒ T extends U ? never : T ⇒ 从联合类型中去除指定的类型；(分配条件类型)_
- Extract<T, U>：** _⇒ T extends U ? T : never ⇒ 与 Exclude 相反，从联合类型中提取指定的类型；_
- NonNullable<T>：⇒ T extends null | undefined ? never : T ⇒ 去除 null 或者 undefined 的类型
- NonNullChecks<T>：排除 T 类型的 Null 和 Undefined 的类型；使用这个类型需要开启一个选项：`"strictNullChecks": true`
- Record<K extends keyof any, T>：作用是生成接口类型，泛型参数分别为接口类型的属性和值

💡注意：type T = keyof any 指代可以作为对象键的属性，即 ⇒ string | number | symbol

# 3.函数类型

- ConstructorParameters<类名>：可以获取类的构造函数的参数组成的元组类型；
- Parameters<函数>：获取函数的参数组成的元组类型；
- ReturnType<函数>：获取函数返回值的类型；
- InstanceType<构造函数/类>：获取构造函数的实例类型；

- ThisParameterType<T(对象/函数)>：可以获取到一个类型中的 this 的类型；
  type Type<T> = T extends (this: infer U, ...args: any[]) => any ? U : unknown

- ThisType<对象>：可以指定对象方法中的 this 指向；在对象字面量中指定 this 的类型
- OmitThisParameter<类型(对象/函数>：用来获取去除函数参数中的 this 类型后的类型；

# 4.字符串类型

- Uppercase<T extends string>：转换字符串字面量类型 T 到大写字母的类型；
- Lowercase<T extends string>：转换字符串字面量类型 T 到小写字母；
- Capitalize<T extends string>：转换字符串字面量类型 T 的第一个字母为大写字母；
- Uncapitalize<T extends string>：转换字符串字面量类型 T 的第一个字母为小写字母；

# 二、自定义工具类型

（——自己造轮子，通常是利用联合类型 never 类型缩小，泛型条件分配类型特性）

  • 条件类型中的类型推断 infer
    type ElementTypeOfArray<T> = T extends (infer E)[] ? E : never
type isNumber = ElementTypeOfArray<number[]> // number
type isNever = ElementTypeOfArray<number> // never

  • keyof关键字：用于提取对象属性名、索引名、索引签名的类型，也可以直接提取函数、类的类型；
    • 表达式上下文中使用typeof，表示获取表达式值的类型；(JS)
    • 类型上下文中使用typeof，则获取变量或者属性的类型；(TS)

  • in关键字：可以用于限定对象属性的范围（只能类型别名定义中使用 in） 返回一个包含限定对象的所有属性类型的新对象类型

  • 映射类型：组合使用 in 和 keyof 创建一个新类型与原类型保持一致性的泛型被称之为映射类型（只能类型别名定义中使用）

    interface A {
  readonly id: number
  name?: string
}

type B<T> = {
  // => 映射类型
  [key in keyof T]: T[key]
  // 可以添加 readonly 可读性   ? 可选性修饰符
  // 可以添加 +、- 前缀表示添加、移除指定修饰符
}

type C = B<A>
// { readonly id: number; name?: string | undefined }

参考例子：判断入参 S 和 T 是否是相同的类型

// 区分出any，此处巧妙使用只有 any 与任意非extends左侧的类型(如0)交叉(如1)才会成为父类型
// 只有 any 满足与任何类型交叉得到的都是 any
type IsAny<T> = 0 extends 1 & T ? true : false // 这里只有传入any才会返回true
type EqualV3<S, T> = IsAny<S> extends true // 注意仍然区分不了大Object 和小object
  ? IsAny<T> extends true
    ? true
    : false
  : IsAny<T> extends true
  ? false
  : [S] extends [T]
  ? [T] extends [S]
    ? true
    : false
  : false
type ExampleV31 = EqualV3<1 | (number & {}), number> // true but false got
type ExampleV32 = EqualV3<never, never> // true
type ExampleV34 = EqualV3<any, any> // true
type ExampleV33 = EqualV3<any, number> // false
type ExampleV35 = EqualV3<never, any> // false
