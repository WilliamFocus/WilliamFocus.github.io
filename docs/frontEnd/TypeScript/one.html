一、TS基础类型

WilliamFocus 2021-04-06 TypeScript

# 1.常用基本类型

  • 数组(Array)：T[] / Array<T> (推荐第一种写法，第二种可能与 jsx 产生语法冲突)
  • 元组(Tuple)： 固定个数元素、每个元素类型未必相同的数组，而不是用对象；（仅做运行时约束，而不能起到真正的限制作用）
  • unknow： 所有类型的父类型，且不可赋值到其他类型，其他类型可以赋值给 unknown
  • any： 可分配到任意类型，忽略类型检查，可以兼容除 never 之外所有的类型（不推荐使用，Any is Hell）
  • void： 返回值为 undefined 或者 null 的函数
  • never： 所有其他类型的子类型，也代表函数中永不会返回值，即不会执行完；
    // 1、在恒为 false 的类型守卫条件判断下，变量的类型将缩小为 never
// 2、（never 是所有其他类型的子类型，所以是类型缩小为 never，而不是变成 never）
const str: string = 'string'
if (typeof str === 'number') {
  str.toLowerCase() // 类型“never”上不存在属性“toLowerCase”.ts(2339)
}

  • 枚举类型(Enums)： 用来表示一个被命名的整型常数的集合（兼具值和类型于一体）详见后面 ⬇️

💡 注意：

  1. 小写格式对应的类型number、string、boolean、symbol是基元类型(原始类型)； 大写类型**Number、String、Boolean、Symbol**则是基本包装对象类型(对象类型)； 原始类型兼容可赋值给对应的对象类型，对象类型不能兼容赋值对应的原始类型 注意两者不等价；（即类似字面量创建的number与new Number()创建的区别） ⇒ 记住尽量不要使用对象类型来注解值的类型！
  2. object类型代表所有非原始类型； Object类型( 简写{} )则代表所有原始类型（null、undefined 除外）和非原始类型； object与Object两者类型互为父子类型； strictNullChecks开启则null、undefined也不能兼容；

# 2.类型断言(as)

（——类似进行仅作用在类型层面的强制类型转换）

const arrayNumber: number[] = [1, 2, 3, 4];

// 第一种：as
const greaterThan2: number = arrayNumber.find(num => num > 2) **as number**;
// 第二种：<T>
const greaterThan2: number = <number>arrayNumber.find(num => num > 2)

// 第三种：
字面量值 + as const 语法结构进行常量断言

// 第四种：非空断言！（**不推荐使用**）
变量、属性的后边添加 '!' 断言操作符，可以用来排除值为 null、undefined 的情况；
推荐使用（类型守卫）代替，这样会更安全，减少运行时bug的出现。

# 3.类型推断

（——编译器可以通过代码所在的上下文推导其对应的类型，无须再声明变量类型）

  • TS 类型标注声明是在变量之后的好处之一就是进行类型推断推导变量类型；
  • 具有初始化值的变量、有默认值的函数参数、函数返回的类型，可以进行类型推断；
let str = 'this is string' // ---> string （字面量类型拓宽）

const str = 'this is string' // ---> 'this is string'

// 这两个定义的变量类型推断出来并不一致
// let定义的类型为string，而const定义类型为'this is string'

# 4.字面量类型

（——字面量类型是集合类型的子类型，它是集合类型的一种更具体的表达）

  • 集合类型：number、string、boolean
  • 字面量类型：1，12，123，‘a’，‘ab’，‘abc’等这些都表示具体类型(即 number、string 的子类型)

字符串字面量类型：即具体的字符串；

数字字面量类型：即具体的数字；

布尔字面量类型：true | false （与 boolean 无区别）；

# 💡 字面量类型拓宽（Literal Widening）

（——字面量子类型转换为父类型）

所有通过 let 或 var 定义的变量、函数的形参、对象的非只读属性，如果满足指定了初始值且未显式添加类型注解的条件，那么它们推断出来的类型就是指定的初始值字面量类型拓宽后的类型

# 💡 其他类型拓宽（Type Widening）

通过let、var定义的变量如果满足未显式声明类型注解且被赋予了 null 或 undefined 值（必须仅限这两个值直接赋值时），则推断出这些变量的类型是 any；

let x = null // 类型拓宽成 any
const y = null // 类型是 null

let z = x // 类型是 null
let z = y // 类型是 null

# 💡 类型收缩（Type Narrowing）

（——通过某些操作将变量的类型由一个较为宽泛的集合缩小到相对较小、较明确的集合）

  • 可以使用类型守卫，也可以通过字面量类型相等值判断（===）或其他控制流语句（包括但不限于 if、三目运算符、switch 分支）将联合类型收敛为更具体的类型；编译器也可以推断出收敛后的具体类型；

# 5.总结

  • let声明的简单类型字面量会拓宽类型；
  • const声明的简单类型字面量会收窄；

注意点：

  1. const声明的对象其属性会自动推断对应的类型，但是可以用as const收窄，让每一个key都是固定类型，会让对象的每个属性变成只读（readonly）
  2. const声明的变量若使用模板字符串进行赋值，若含有变量，则会拓宽为string类型，否则为收缩的字符串字面量类型；
