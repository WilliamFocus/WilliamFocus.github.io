二、TS函数与类

WilliamFocus 2021-04-15 TypeScript

# 一、函数类型

// 方式一  显式指定
const add = (a: number, b: number): number => {
     return a + b;
}
    👇
// 方式二   抽离复用
interface Entity {   // 或 type Entity = {
    add: (a: number, b: number) => number; // 注意返回值必须是**类型**
    del(a: number, b: number): number; // 同对象属性简写语法
		<T extends number, N>(a: T, b: N): number; // 实际上一一对应映射函数参数和返回值的类型
}
// 注意：若显式添加函数返回值类型不能为undefined，必须为 void 或 any

// 可定义 Generator 函数类型
type AnyType = boolean;
type AnyReturnType = string;
type AnyNextType = number;
function *gen(): Generator<AnyType, AnyReturnType, AnyNextType> {
  const nextValue = yield true; // nextValue 类型是 number，yield 后必须是 boolean 类型
  return `${nextValue}`; // 必须返回 string 类型
}

从类型层面看，我们也可以通过类型推断加工计算入参的类型，并返回新的类型；

# 1、可选参数：?: T

  • ?: 不等于 ≠ 联合类型参数 T | undefined ?: 表示参数可以缺省、可以不传；联合类型不可缺省且类型必须是 T 或者 undfined
  • 注意可选参数后不允许出现必选参数；

# 2、默认参数：x = T

会自动推断出了 x 的类型为 ⇒ T | undefined

# 3、剩余参数：

function sum(...nums: number[]) {...} function sum(...nums: (number | string)[]) {...}

# 4、this 类型：

function say(this: T, name: T) {...}

  1. 函数参数的第一个位置可以显式注解函数内部 this 的类型；
  2. this 类型不会占据函数参数位置，编译时“伪形参” this 会被抹掉；
  3. 若 this 显式指定为 Window 类型，则调用时也必须手动添加 window，因为隐式调用时函数的调用者类型默认为 void，TS 不清楚调用者（符合 JS 严格模式下函数 this 默认指向 undefined）

# 5、函数重载（Function Overload）

（——TypeScript 会从上到下查找函数重载列表中与入参类型匹配的类型，并优先使用第一个匹配的重载定义，即描述参数与返回值类型约束关系的函数类型）

  • 可以多次声明相同函数的描述函数类型列表，当实现该函数会自动匹配形参重载函数类型的定义
  • 💡**注意：**继承的子类型也会符合并匹配到父类型；
interface P1 {
    name: string;
}
interface P2 extends P1 {
    age: number;
}

function convert(x: P1): number;
function convert(x: P2): string;

const x1 = convert({ name: "" } as P1) // => number
const x2 = convert({ name: "", age: 18 } as P2) //number->可以互换一下上面类型列表声明顺序
//
// **注意：**此处使用断言以保证x2的参数类型(P2)是继承P1的，
**//** 否则会对比对象的每一个属性，从而匹配到第二个函数类型列表；

# 二、类-类型

// 一、基类(父类)
class Animal {
  weight: number // ---> 声明构造函数中实例属性上的属性类型

  // 访问修饰符
  public firstName: string // ---> 修饰公共公有的可访问属性或方法
  protected lastName: string = 'Stark' // ---> 修饰仅类自身及子类中可访问受保护属性或方法
  private fullName: string // ---> 修饰的是仅在同一类中可见的私有的属性或方法

  // 只读修饰符 readonly
  public readonly firstName: string // ---> 修饰公共且只读的属性或方法(有public则需要先写)

  // 存取修饰符
  get myLastName() {}
  set myLastName(name: string) {}

  // 静态属性修饰符 // ---> 注意: 需要显式注解 this 类型才可以在静态方法中使用 this
  static displayName = 'MyArray'

  constructor(size: number) {
    this.weight = size
  }
}

// 二、派生类(子类)
class Dog extends Animal {
  name: string
  constructor(name: string) {
    super(10) // ---> 子类构造函数必须实现super调用父类构造函数
    this.name = name
  }
}

// 三、抽象类(对类的抽象,不能实例化)
abstract class Adder {
  abstract x: number // abstract 定义抽象属性(派生类必须实现)
  abstract y: number
  abstract add(): number
  displayName = 'Adder' // ---> 可以被派生类继承和使用的非抽象属性
  addTwice(): number {
    return (this.x + this.y) * 2
  }
}

💡 注意：

  1. 类的类型和函数类似，即在声明类的时候，其实也同时声明了一个特殊的类型（确切地讲是一个接口类型），这个类型的名字就是类名，表示类实例的类型；在定义类的时候，我们声明的除构造函数外所有属性、方法的类型就是这个特殊类型的成员。
  2. TypeScript 中定义类的私有属性仅仅代表静态类型检测层面的私有。如果强制忽略类型的检查错误，转译且运行 JavaScript 时依旧可以获取到受保护或私有属性，因为 JavaScript 并不支持真正意义上的私有属性。(但是目前 JS 类修饰符已有提案，将来或可实现)
